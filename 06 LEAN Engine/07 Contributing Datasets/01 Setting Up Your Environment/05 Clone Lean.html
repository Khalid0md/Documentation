Follow these steps to get Lean into Visual Studio Code Dev Containers

  1. Fork Lean repository.
  2. Clone the Lean repository.
    3. $ git clone https://github.com/username/Lean.git
  3. Open the Lean directory in Visual Studio Code
    1. Change the mounts property in the devcontainer.json to set the `source` of the data and the data source project.
      "mounts": [ "source=C:/Users/username/Lean/Data,target=/Data,type=bind,consistency=cached", "source=C:/Users/username/Lean.DataSource.<vendorNameDatasetName>,target=/Lean.DataSource.<vendorNameDatasetName>,type=bind,consistency=cached" ]
    2. Open Development Container, either
      1. Select "Reopen in Container" from pop up box
      2. Ctrl+Shift+P (Command Palette) and select "Remote-Containers: Rebuild and Reopen in Container"

      The Visual Studio Code's extensions will load, the OmniSharp extension will analyse the project, and ask to restore the dependecies.

    3. Change the data-folder property in the /Lean/Launcher/config.json to the mounted data directory.
      "data-folder": "/Data/",
    4. Run Lean to assert successful installation
      1. Open the Run and Debug tab (Ctrl+Shift+D).
      2. Select Launch and press F5.

      Visual Studio Code will compile Lean and execute the C# version of BasicTemplateFrameworkAlgorithm.
